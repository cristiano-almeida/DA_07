📖 Conceitos Fundamentais de POO

🎯 O que é POO?

A Programação Orientada a Objetos é um paradigma que organiza o código em objetos que contêm dados (atributos) e comportamentos (métodos).

📦 Classe x Objeto

ConceitoDefiniçãoAnalogia
Classe Molde que define atributos e métodos Planta de uma casa
Objeto Instância da classe (ocupam memória) A casa construída
Atributo Características do objeto Cor, tamanho, quantidade de quartos
Método Ações que o objeto pode realizar Abrir porta, ligar luz
📚 TRÍADE DIDÁTICA:
  • CONCEITO: O que é? (definição)
  • OBJETIVO: Para que serve? (utilidade)
  • MÉTODO: Como se faz? (implementação)

🔒 Modificadores de Acesso em Java

Os modificadores de acesso controlam quem pode ver e quem pode usar atributos e métodos.

ModificadorO que significaOnde pode ser acessado
(nenhum)
default/package-private		 Acessível dentro do mesmo pacote 📁 Somente classes no mesmo pacote
public Público - acessível de qualquer lugar 🌍 Qualquer classe, qualquer pacote
private Privado - acessível apenas dentro da classe 🔒 Somente dentro da própria classe
protected Protegido - acessível no pacote e subclasses 📁 Pacote + herança
⚠️ IMPORTANTE: Quando você NÃO coloca modificador (default), o atributo/método é acessível apenas dentro do mesmo pacote!

🎯 Exemplo prático dos modificadores

CONCEITO: Modificadores de acesso controlam a visibilidade.
OBJETIVO: Proteger dados e organizar o código.
MÉTODO: Usar private para atributos e public para métodos de acesso (getters/setters). 
package tema07;

public class ExemploModificadores {
    // 1. SEM MODIFICADOR (default) - acessível no pacote
    String nomeDefault = "Acessível no pacote";

    // 2. PUBLIC - acessível em qualquer lugar
    public String nomePublic = "Acessível em qualquer lugar";

    // 3. PRIVATE - acessível apenas dentro desta classe
    private String nomePrivate = "Acessível apenas aqui";

    public void exibir() {
        // Todos os atributos são acessíveis DENTRO da classe
        System.out.println(nomeDefault);
        System.out.println(nomePublic);
        System.out.println(nomePrivate);  // OK!
    }
}
💡 BOA PRÁTICA: Atributos geralmente são private (encapsulamento). Métodos de acesso são public (getters/setters).

📌 Exemplo 1: Classe Aluno (sem encapsulamento)

📚 CONCEITO: Atributos públicos e método de exibição.
🎯 OBJETIVO: Mostrar como é simples criar uma classe, mas os dados ficam expostos.
⚡ MÉTODO: Atributos public ou sem modificador (default).
🔍 OBSERVAÇÃO: Neste exemplo, os atributos estão sem modificador (default) ou públicos. Isso permite acesso direto de qualquer lugar no mesmo pacote. 
package tema07;

public class Exemplo1Aluno {
    // ATRIBUTOS SEM MODIFICADOR (default) - acessíveis no pacote
    String nome;
    int idade;
    double nota;

    // MÉTODO - exibe os dados
    public void exibirDados() {
        System.out.println("Nome: " + nome);
        System.out.println("Idade: " + idade);
        System.out.println("Nota: " + nota);
    }

    // MÉTODO MAIN para teste
    public static void main(String[] args) {
        // Criando objeto
        Exemplo1Aluno aluno = new Exemplo1Aluno();

        // ACESSO DIRETO aos atributos (permitido porque estão no mesmo pacote)
        aluno.nome = "João Silva";
        aluno.idade = 20;
        aluno.nota = 8.5;

        // Exibindo
        aluno.exibirDados();

        // ⚠️ TESTE: Alterando diretamente sem validação
        System.out.println("\n--- TESTE: Alterando nota diretamente ---");
        aluno.nota = -10;  // PERMITIDO, mas não faz sentido!
        aluno.exibirDados();
    }
}
📤 Saída:
Nome: João Silva
Idade: 20
Nota: 8.5

--- TESTE: Alterando nota diretamente ---
Nome: João Silva
Idade: 20
Nota: -10.0
⚠️ PROBLEMA: Como os atributos são acessíveis diretamente, qualquer um pode colocar valores inválidos (ex: nota negativa) sem nenhum controle!

📌 Exemplo 2: Classe Aluno (com encapsulamento)

📚 CONCEITO: Atributos privados e métodos públicos de acesso (getters/setters).
🎯 OBJETIVO: Proteger os dados, permitindo apenas valores válidos.
⚡ MÉTODO: Atributos private, getters/setters public com validações.
🔍 OBSERVAÇÃO: Neste exemplo, os atributos são PRIVADOS. O acesso é feito apenas por métodos públicos que validam os valores antes de atribuir. 
package tema07;

public class Exemplo2Aluno {
    // ATRIBUTOS PRIVADOS - NINGUÉM acessa diretamente!
    private String nome;
    private int idade;
    private double nota;

    // GETTERS - para LER os valores
    public String getNome() {
        return nome;
    }

    public int getIdade() {
        return idade;
    }

    public double getNota() {
        return nota;
    }

    // SETTERS - para DEFINIR os valores (com validação!)
    public void setNome(String nome) {
        if (nome != null && !nome.trim().isEmpty()) {
            this.nome = nome;
        } else {
            System.out.println("❌ Nome inválido!");
        }
    }

    public void setIdade(int idade) {
        if (idade >= 0 && idade <= 150) {
            this.idade = idade;
        } else {
            System.out.println("❌ Idade inválida! (0-150)");
        }
    }

    public void setNota(double nota) {
        if (nota >= 0 && nota <= 10) {
            this.nota = nota;
        } else {
            System.out.println("❌ Nota inválida! (0-10)");
        }
    }

    // MÉTODO para exibir os dados
    public void exibirDados() {
        System.out.println("Nome: " + nome);
        System.out.println("Idade: " + idade);
        System.out.println("Nota: " + nota);
    }

    // MÉTODO MAIN para teste
    public static void main(String[] args) {
        Exemplo2Aluno aluno = new Exemplo2Aluno();

        // Usando SETTERS - valores válidos
        System.out.println("--- TESTE 1: Valores válidos ---");
        aluno.setNome("Maria Santos");
        aluno.setIdade(22);
        aluno.setNota(9.5);
        aluno.exibirDados();

        // Usando SETTERS - valores INVÁLIDOS (serão bloqueados!)
        System.out.println("\n--- TESTE 2: Valores INVÁLIDOS ---");
        aluno.setNota(-5);   // ❌ Bloqueado!
        aluno.setIdade(200); // ❌ Bloqueado!
        aluno.setNome("");   // ❌ Bloqueado!

        System.out.println("\n--- Dados permanecem intactos ---");
        aluno.exibirDados();
    }
}
📤 Saída:
--- TESTE 1: Valores válidos ---
Nome: Maria Santos
Idade: 22
Nota: 9.5

--- TESTE 2: Valores INVÁLIDOS ---
❌ Nota inválida! (0-10)
❌ Idade inválida! (0-150)
❌ Nome inválido!

--- Dados permanecem intactos ---
Nome: Maria Santos
Idade: 22
Nota: 9.5
✅ VANTAGEM: O encapsulamento protege os dados! Valores inválidos são bloqueados antes de serem atribuídos.

⚖️ Comparação: Exemplo 1 vs Exemplo 2

CaracterísticaExemplo 1 (sem encapsulamento)Exemplo 2 (com encapsulamento)
Modificador dos atributos default (sem modificador) private
Acesso aos atributos Direto: aluno.nota = -10; Via setter: aluno.setNota(-10); (bloqueia)
Validação de dados ❌ Nenhuma validação ✅ Validação nos setters
Segurança dos dados ❌ Dados expostos e vulneráveis ✅ Dados protegidos
Controle de valores ❌ Qualquer valor é aceito ✅ Apenas valores válidos
Facilidade de manutenção ❌ Difícil (dados espalhados) ✅ Fácil (centralizado nos setters)
Complexidade ✅ Simples ⚠️ Um pouco mais de código
💡 CONCLUSÃO: O encapsulamento vale o esforço extra! Ele protege seus dados e torna o código mais seguro e confiável.

🎯 Testes Didáticos para Fazer

🧪 Teste 1: No Exemplo 1, tente colocar aluno.idade = -5;. O que acontece?
Resposta: O código compila e executa, mas a idade fica negativa (inválida!).
🧪 Teste 2: No Exemplo 2, tente colocar aluno.setIdade(-5);. O que acontece?
Resposta: O setter bloqueia e exibe: "❌ Idade inválida! (0-150)".
🧪 Teste 3: No Exemplo 2, tente acessar diretamente aluno.nota = 10;. O que acontece?
Resposta: Erro de compilação! O atributo é private, não pode ser acessado diretamente.
🧪 Teste 4: No Exemplo 1, mude os atributos para public. O que muda?
Resposta: Fica acessível de qualquer lugar (outros pacotes também). Continua sem validação.
